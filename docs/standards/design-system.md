# Design system standard

## Canonical sources

- **Repo docs are canonical** for design-system rules, decisions, and agent guidance.
- **GitHub is canonical** for tracking design work and change history.
- **SQLite under `.devworkspace/` is retrieval/index state only**, not truth.

## Rules

- Optimize for clarity before novelty.
- Use one term per concept and keep labels consistent across UI and docs.
- Show clear loading, empty, error, and success states.
- Make interactive states obvious: focus, hover, active, disabled.
- Default to accessible structure, readable contrast, and keyboard-usable flows.
- Do not imply a local issue tracker or hidden work queue in the product.

## Language rule

If the work is tracked in GitHub, say GitHub. If guidance lives in repo docs, point to repo docs. Do not blur the source of truth.
